Add New 3rd Party Library

Depending on the nature of the library, they are mostly globally available once loaded. Some might require manual instantiations or only to be instantiated when needed. Hence some level of customization and understanding is required to keep this web app optimized and organized.

Please follow the steps below to understand and learn the process of adding a standard new 3rd party javascript library like `moment.js` and how to use it.

  1. Use brower to add the javascript library to the vendor folder. Or simply extract the full/release folder of the library into the vendor folder. Create its relevant folder though just like other ones.

  2. Open up `config.js` and configure the path for requireJS to load.

    "moment": cdnURL+"vendor/newlibrary/min/newlibrary.min"

  3. Then open up `app.js` file and include the library in the second level require loading argument.

    require([ "canjs-ejs", "foundation", "moment", "moment-lang", "dg-components", "fastclick", "newlibrary", // INSERT BEFORE COMMON "common"], function ( _can_ejs_, _foundation_, _moment_, _moment_lang_, _components_, _fastclick_, _newlibrary_, // INSERT BEFORE COMMON _common_ )) {...}

    The reason that it should be inserted before common is to help with ordering of loading in case this library require instantiations.

  4. If the library is used everywhere and needs instantiations, do it in the success block of the second level `require()` inside `app.js` or `common()` function inside `common.js`. Decide where it should fit as you see fit. Otherwise, instantiate as needed whenever, wherever.

* Grunt publishes all folders inside `vendor` into `publish` automatically, so no modification of the grunt's configuration is required.
