How To

What is a Component?

A component in this DG Web App is simply a canJS `Control` built specifically for a particular user function. It can be a widget like the `auto_address` search input. It could also be a more behind the scene function like `page_slider`.

This is not a canJS `component`!

Basically, anything that can be reused (or placed as a global function in `common.js`) can be none the less built as a component. The purpose is to allow cleaner management of things that are more complicated for better maintainability and extensibility.

Add New Component

Please follow the steps below to understand and learn the process of adding a new standard `js/component` component and how to use it.

Snake Game Component
  1. Create a file named correctly (`snake_game.js`) after your component name under `js/components/`

  2. Place in the empty component structure as follows:

    _global['_components']['SnakeGame'] = { id: 'SnakeGame', instantiate: function () { initDictionary(snakeGameStrEN); this.Control = can.Control({ init: function () { // Init the functionalities here } }); }, create: function (str, element) { return new this.Control(element, { errorMessage: str }); } };

    Please note the naming of the component `SnakeGame` as this is very important for lazy loading by the DG Web App.

  3. Do what you have to do to build this control just like any other canJS controls that you build.

  4. If you need a string file to be attached, then simply create a translation file just like how features are done and have it suffixed with `_str.en.js`. Use the right naming of the dictionary for loading. (`snakeGameStrEN`) Ideally, try to prefix the keys with a different/unique letter to this component so that it does not interfere with other files. (`SNAKE` would be a good choice.)

    If you take a look at `popup_str.en.js`, you can see that keys are prefixed with `POP`.

  5. Components are usually shared functionality that can be used anywhere and anytime. Also, they are compiled all in to one file `js/components/components.min.js` to reduce file count. Edit `Gruntfile.js` and include the component file under the `'js/components/components.min.js'` key block. Remember, always include the string file first!

  6. Component controls are instantiated automatically so that they are loaded in memory ready for use. To use this component:

    _c("SnakeGame").create("I AM SNAKE!", someDivHolder);
  7. Done!

Some components like the popup component is customized. Its `create()` function uses a singleton pattern.
