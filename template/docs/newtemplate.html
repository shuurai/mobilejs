How To

EJS/TPL Templates

All tpls are common snippets in the form a EJS with the exception of `popupEJS.tpl`, and gets compiled as part of the "index.html" file when built using grunt. Tpl files in the `tpl/*` folder gets picked up and compiled automatically. Tpls are usually used for things that are common and needed all the time, and for other things that is better to be loaded lazily or dynamically, they are stored in the `ejs/*` folder and seperated by their type.

To understand more about EJS templates, please see the official [embeded.js] website and [can.js] about how they are integrated.

How to Add a New `.TPL` Template

TPLs should mostly contain the EJS script tags. Imagine this is part of an html file just split up. The `popupEJS` is a peculiaer case due to the way foundation's reveal canvas is designed. TPLs are generally html embedded EJS templates.

Please follow the steps below to understand and learn the process of adding a new standard `tpl` file and how to use it.

  1. Create the tpl file inside `tpl/common/` and make sure it is suffixed with `EJS`.
  2. Make sure the EJS name in the script tag `<script type='text/ejs' id='newTplEJS'>` is the same as the file name to be consistent.
  3. The file will be automatically picked up by grunt and is now available for use throughout the system.

A key concept for most tpl widgets is that, a `content`, `form` and `data` attributes are available for use in any EJS script tags. These objects are usually passed in at the time of instantiation. Of course depending on the nature of this tpl widget, the passed in attributes can be different.

How to Add a New `.EJS` Template

`.ejs` files are specific to features. They are external EJS templates for dynamic and as required loading.

Please follow the steps below to understand and learn the process of adding a new standard `ejs` file and how to use it.

  1. Create the EJS file inside `ejs/[featurename]/` and make sure it has the same name as the corresponding feature's page.
  2. Make sure the EJS file does not have the the script tag like `<script type='text/ejs' id='newTplEJS'>` as it is not required for externally loaded files.
  3. The file will be coded in its relevant feature to be loaded when required.

EJS templates are dependant on feature implementations so please read about features for more information.
